# v5 修复任务：基本面消息卡片的条件渲染太严

## 背景
上一轮（v4）你（AI）只改了 `/api/market/quotes/route.ts` 修复了 JSONP 解析，走势和顶部净值预计已修好。但**基本面消息卡片仍不会显示**，用户在详情页看不到「基本面消息」这个卡片，走势下面直接跳到"提醒规则"。

你在 v4 回复里承认：`page.tsx` 第 346-348 行的挂载代码是这样的：
```jsx
{item.type === 'fund' && (
  <FundamentalsNews fundCode={item.code} fundName={item.name} />
)}
```

## 根本问题

用户 watchlist 里的 5 只标的都是基金/ETF（华夏沪深300增强C、国联安半导体ETF联接C、招商中证白酒C、华夏软件龙头C、华安黄金ETF联接C），但是：
1. 你不知道 watchlist 表里 `type` 字段的**实际取值**是什么——可能是 `null`、`''`、`'etf'`、`'index_fund'`，也可能只有一部分记录是 `'fund'`
2. `item.type === 'fund'` 是**极其严苛的相等判断**，只要 type 不是精确的字符串 `'fund'`，卡片就永远不显示

## 你要做的（一次做完，别再糊弄）

### 步骤 1：查一次 Supabase watchlist 实际 type 值
用 Supabase 客户端执行等价的 SQL：
```sql
select id, code, name, type from watchlist limit 20;
```
在 AI 回复里贴出返回结果（type 字段的真实取值），让我们看到 type 字段到底是什么。

### 步骤 2：放宽条件渲染
按以下优先级选一个方案，改 `src/app/detail/[id]/page.tsx` 第 346-348 行：

**方案 A（推荐，最宽容）**：完全去掉 type 条件，任何 watchlist item 都显示基本面消息卡片
```jsx
<FundamentalsNews fundCode={item.code} fundName={item.name} />
```

**方案 B（次选，兼容多种基金类型）**：把等号改为白名单
```jsx
{['fund', 'etf', 'index_fund', 'lof', null, undefined, ''].includes(item.type as any) && (
  <FundamentalsNews fundCode={item.code} fundName={item.name} />
)}
```

**方案 C（保底）**：把 `item.type === 'fund'` 改成 `item.type !== 'stock'`（假设未来有股票类型）

任选一个，只要保证用户当前 5 只基金都能显示卡片就行。**推荐方案 A**，因为基本面消息对基金/ETF/股票都有价值，没必要屏蔽。
